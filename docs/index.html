Product role

Blockwright is the control layer between AI models and a Minecraft Java Edition world.

It turns player or operator requests into explicit, auditable Minecraft operations. The assistant can read context, choose supported tools, execute through server-side APIs, and report what changed.

Unified request entrances

Route requests from /bw, Web text, voice input, Matrix/DingTalk, or local scripts into the same controlled assistant runtime.

Java Edition deployment

The primary runtime is a Fabric mod for Java Edition 1.21.x, single-player saves, and LAN-opened worlds. Paper remains available for standalone server deployments.

Controlled execution

Actions pass through typed controller tools: player/world reads, approved command operations, inventory actions, block placement, blueprints, and task queues.

Records and verification

Build plans are saved before execution. The Minecraft side reads world state back after placement so records, reports, and actual blocks stay aligned.

Supported models

Supported AI backends are configured from the Web settings page.

Model credentials stay in local configuration. Once selected, the backend powers the same Minecraft command, Web, voice, and chat-tool entrances.

Codex CLI

Local agent runtime for controller-integrated planning and tool execution.

OpenAI

Hosted model backend configured through local settings and environment-safe credentials.

DeepSeek

Hosted model backend for natural-language planning and tool selection.

Doubao

Volcengine/Doubao backend for deployments that need strong Chinese-language support.

Gemini

Google Gemini backend for assistant requests and planning.

Commands and entrances

Minecraft command and tool entrances backed by production execution controls.

Operators and players submit intent through supported entrances. Blockwright converts that intent into tool calls, queued actions, command operations, and verification reports.

/bw <request>

Main Minecraft command

Submit operational requests for world inspection, inventory actions, time and weather changes, supported commands, build placement, and structure edits.

/bw web

Open Web UI

Print the local Web address in Minecraft chat for player binding, provider configuration, text chat, voice input, and runtime settings.

Web text and voice

Browser entrance

Use the browser as an operator console for text or voice requests routed to the configured Minecraft player.

Matrix, DingTalk, local scripts

Chat-tool entrances

Connect external chat systems or local automation scripts when Minecraft operations need to originate outside the game window.

Deployment flow

Install, configure, and operate Blockwright from Minecraft or the Web console.

The runtime is designed for Java Edition worlds with a local controller, explicit model configuration, and Minecraft-side execution through the Fabric mod.

Prepare Minecraft

Use Minecraft Java Edition 1.21.x with Fabric Loader 0.16.14 or newer.

Put mods in mods/

Install Fabric API for your 1.21.x version, then put blockwright-fabric-*.jar in the same mods/ folder.

Choose an AI model

Run /bw web, open the Web page, set your Minecraft username, and choose Codex CLI, OpenAI, DeepSeek, Doubao, or Gemini.

Operate Minecraft

Submit requests through /bw, Web text, voice, or a connected chat tool. The controller routes them through typed tools and records execution results.
