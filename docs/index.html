What it does

Blockwright turns Minecraft requests into controlled in-game actions.

It is meant for local Minecraft Java Edition play first: ask inside the world, let the assistant plan, and let the Fabric mod place or verify the result through server-side APIs.

Talk in game

Use /bw to ask for items, time changes, builds, or edits without leaving the Minecraft world.

Fabric local worlds

The main path is a Java Edition/Fabric mod for single-player saves and LAN-opened worlds, so maps do not need to move to a Paper server.

Build with records

Build plans are saved as blueprints and records before the same block list is sent to Minecraft for placement.

Edit safely

For existing builds, Blockwright scans nearby world blocks and matches saved build records before changing the structure.

Try it locally

Start the controller, install the Fabric mod, then send a request in game.

Blockwright is early but runnable. The simplest flow is local controller plus Fabric mod in your current Minecraft 1.21.8 game directory.

Start the controller

Run ./scripts/run-web.sh from the repository root. The default local URL is http://127.0.0.1:8765/web.

Install into Java Edition/Fabric

Run make to build and install the mod into the current Java Edition game directory, or specify one with make GAME_DIR=....

Use /bw in the world

Try requests like /bw give me a diamond sword or /bw build a small cabin in front of me.

How it works

The mod touches Minecraft; the controller handles planning, records, and integrations.

That split keeps Minecraft execution concrete: blocks are placed through Fabric/Paper world APIs, and the controller keeps the assistant, queue, blueprints, and verification records outside the game.
