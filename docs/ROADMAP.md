# 路线图

## 第 1 阶段：跑通闭环

- Rust controller 提供 API。
- Paper 插件可以发送游戏内命令到 controller。
- 外部机器人可以通过 HTTP 创建 Minecraft 任务。
- 插件能发物品、放置示例蓝图。

## 第 2 阶段：蓝图能力

- 增加插件扫描方块区域的接口。
- 支持把当前建筑保存成蓝图。
- 支持蓝图列表、蓝图详情、蓝图删除。
- 支持按玩家坐标识别附近已放置蓝图。

## 第 3 阶段：Codex 规划

- controller 调用本地 `codex` 命令。
- 让 Codex 输出结构化 JSON，不直接输出自由文本。
- 对 Codex 输出做校验，校验通过后才下发给插件。

## 第 4 阶段：图片复刻

- 接收图片 URL 或上传文件。
- 图片分析后生成候选蓝图。
- 先保存蓝图，再由玩家确认是否建造。

## 第 5 阶段：多机器人接入

- Telegram webhook。
- Discord bot。
- 企业微信/飞书机器人。
- 每个机器人都只调用 controller，不直接接 Minecraft。
