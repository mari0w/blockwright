# Blockwright：面向 Minecraft Java 版的生产级 AI 控制层

如果你经常玩 Minecraft Java 版，大概会遇到这样的时刻：脑子里已经知道要做什么，但还得停下来把想法翻译成命令。

- 想知道自己手上拿的是什么，或者面前这块方块是什么；
- 想快速执行物品、效果、时间、天气或模式相关操作，但不想翻命令表；
- 想描述一个建筑想法，再一步步继续调整；
- 想改造眼前已有结构，而不是手动替换每一块方块；
- 想让朋友在局域网世界里也能直接通过聊天发真实需求。

Blockwright 做的事情很直接：把这些 Minecraft 需求变成自然语言指令，再让 Minecraft AI 助手通过受控工具、命令操作、构建记录和校验报告执行。

你可以在游戏里输入：

```text
/bw 扫描我看着的地方，告诉我这是什么方块
/bw 把时间调到白天并停止下雨
/bw 帮我盖一个有窗户、床、灯和可进入门的小屋
/bw 把我面前这面墙换成石砖
```

然后 Blockwright 会把玩家的意图交给 AI 助手理解，再通过 Fabric 模组读取世界、执行动作、放置或改造方块，并返回执行结果。

## Blockwright 是什么？

Blockwright 是面向 Minecraft Java 版的生产级 AI 控制层。

它的主安装方式是 Fabric 模组，适合 Java 版 1.21.x、单人存档和局域网开放世界。玩家不需要把原来的地图迁移到 Paper 服务端，也不需要为了使用 AI 助手改变自己的游玩方式。

装好后，你可以通过几种方式和它说话：

- 在 Minecraft 聊天栏里输入 `/bw ...`
- 打开 Web 页面直接打字
- 在 Web 页面按住麦克风说话
- 通过 Element/Matrix 房间发消息
- 通过钉钉机器人发消息
- 通过本地命令或脚本接入其他工具

它不只是聊天回复、发物品或建一个小建筑。Blockwright 给 Minecraft AI 助手一层受控工具，让它能读取世界上下文、执行支持的命令操作、建造、改造，并校验世界里的真实结果。

## 它能做什么？

### 1. 读取玩家和世界上下文

Blockwright 可以先读取真实游戏状态，而不是只靠聊天文字猜：

```text
/bw 我手上拿的是什么
/bw 看一下我的背包
/bw 扫描我附近的方块
```

这些上下文会被用于改造、建造、物品需求和普通游戏操作。

### 2. 执行受控游戏动作

助手可以通过 controller 和 Minecraft 执行端运行支持的动作：

```text
/bw 给我一组火把
/bw 把时间调到白天
/bw 别下雨了
/bw 给我夜视
/bw 切到创造模式
```

这些常见操作可以用自然语言表达，但执行时仍然走明确动作类型和服务端 API。

### 3. 带记录和校验地建造

对于建造需求，Blockwright 会先保存计划，再交给 Minecraft 执行：

```text
/bw 在我面前建一个带窗户和床的房间
/bw 建一个简单的树屋，入口要能走上去
/bw 从这里到门口加一条有照明的小路
```

蓝图和构建记录由 controller 保存，Fabric 模组通过服务端世界 API 放置方块，执行后还会回读世界方块做校验。

### 4. 改造眼前的东西

除了新建，它也可以处理一些“改造”类需求：

```text
/bw 把这个房子的窗户换成蓝色玻璃
/bw 把我面前这面墙换成石砖
/bw 把屋顶加高一点
```

这种交互更接近“你在游戏里指着一个地方，然后告诉助手要怎么改”。

### 5. 继续对话

Blockwright 会围绕当前玩家和上下文继续处理需求。你可以连续说：

```text
/bw 帮我盖一个木屋
/bw 换大一点
/bw 加几盏灯
/bw 把屋顶换成深色木头
```

这种方式比一次性写完整需求更自然，也更接近真实玩家的建造流程。

## 怎么开始使用？

玩家侧流程：

1. 准备 Minecraft Java Edition 1.21.x。
2. 安装 Fabric Loader 0.16.14 或更新版本。
3. 在当前游戏目录的 `mods/` 文件夹里放入：
   - 对应 1.21.x 游戏版本的 Fabric API
   - `blockwright-fabric-*.jar`
4. 启动 Minecraft，进入世界。
5. 在游戏聊天栏输入：

```text
/bw web
```

打开显示出来的 Web 地址后，填写自己的 Minecraft 用户名，再选择一个 AI 模型。

支持的模型后端：

- Codex CLI
- OpenAI
- DeepSeek
- 豆包 Doubao
- Gemini

配置完成后，就可以在 Web 页面、语音、游戏内 `/bw` 或聊天工具里发送需求。

## 为什么这件事重要？

Minecraft 的魅力在于自由创造，但很多重复操作本质上只是准备工作或命令语法。

读取附近状态、拿物品、调时间、清雨、搭一个房间、替换一片墙、给房间补灯，这些事情都很常见。Blockwright 把这些操作变成一套受控的助手工作流，让玩家和运营把注意力放回正在创造的世界，同时保留明确、可追踪的执行过程。

它适合：

- 在单人存档里使用带世界上下文的 AI 助手；
- 局域网世界里让朋友通过聊天发需求；
- 在创造模式里创建并持续改造建筑；
- 服主或运营把 Minecraft 接到 Web、语音或聊天工具里；
- 需要 AI 操作 Minecraft、且希望结果可记录可校验的玩家和团队。

## 一句话总结

Blockwright 是一个能听懂自然语言、也能调用受控工具的 Minecraft AI 助手：你提出操作意图，它把重复、繁琐、命令化的工作变成游戏里的实际动作。

它面向 AI 操作 Minecraft 的工作流：读取上下文、执行支持动作、建造、改造、记录，并校验结果。
