Blockwright：面向 Minecraft Java 版的生产级 AI 控制层

这是一篇可以直接分享的产品概览：讲清楚 Blockwright 的能力、支持的模型、命令入口和 Minecraft 执行方式。

如果你经常玩 Minecraft Java 版，大概会遇到这样的时刻：脑子里已经知道要做什么，但还得停下来把想法翻译成命令。

  • 想知道自己手上拿的是什么，或者面前这块方块是什么；
  • 想快速执行物品、效果、时间、天气或模式相关操作，但不想翻命令表；
  • 想描述一个建筑想法，再一步步继续调整；
  • 想改造眼前已有结构，而不是手动替换每一块方块；
  • 想让朋友在局域网世界里也能直接通过聊天发真实需求。

Blockwright 做的事情很直接：把这些 Minecraft 需求变成自然语言指令，再让 Minecraft AI 助手通过受控工具、命令操作、构建记录和校验报告执行。

/bw 扫描我看着的地方，告诉我这是什么方块
/bw 把时间调到白天并停止下雨
/bw 帮我盖一个有窗户、床、灯和可进入门的小屋
/bw 把我面前这面墙换成石砖

然后 Blockwright 会把玩家的意图交给 AI 助手理解，再通过 Fabric 模组读取世界、执行动作、放置或改造方块，并返回执行结果。

Blockwright 是什么？

Blockwright 是面向 Minecraft Java 版的生产级 AI 控制层。

它的主安装方式是 Fabric 模组，适合 Java 版 1.21.x、单人存档和局域网开放世界。玩家不需要把原来的地图迁移到 Paper 服务端，也不需要为了使用 AI 助手改变自己的游玩方式。

装好后，你可以通过几种方式和它说话：

  • 在 Minecraft 聊天栏里输入 /bw ...
  • 打开 Web 页面直接打字
  • 在 Web 页面按住麦克风说话
  • 通过 Element/Matrix 房间发消息
  • 通过钉钉机器人发消息
  • 通过本地命令或脚本接入其他工具

它不只是聊天回复、发物品或建一个小建筑。Blockwright 给 Minecraft AI 助手一层受控工具，让它能读取世界上下文、执行支持的命令操作、建造、改造，并校验世界里的真实结果。

它能做什么？

读取玩家和世界上下文。 询问手持物、查看背包，或在操作前扫描附近方块。

执行受控游戏动作。 发物品、调时间、停雨、加效果、切模式，或通过明确动作类型执行支持的命令。

带记录和校验地建造。 建造请求会先保存成蓝图和构建记录，再由 Fabric 模组放置方块并回读世界校验。

改造眼前的东西。 比如 /bw 把这个房子的窗户换成蓝色玻璃，更接近“你在游戏里指着一个地方，然后告诉助手要怎么改”。

继续对话。 你可以先说“盖一个木屋”，再说“换大一点”“加几盏灯”“把屋顶换成深色木头”。

怎么开始使用？

  1. 准备 Minecraft Java Edition 1.21.x。
  2. 安装 Fabric Loader 0.16.14 或更新版本。
  3. 在当前游戏目录的 mods/ 文件夹里放入对应 1.21.x 游戏版本的 Fabric API 和 blockwright-fabric-*.jar。
  4. 启动 Minecraft，进入世界。
  5. 在游戏聊天栏输入 /bw web。

打开显示出来的 Web 地址后，填写自己的 Minecraft 用户名，再选择一个 AI 模型。支持的模型后端包括 Codex CLI、OpenAI、DeepSeek、豆包 Doubao 和 Gemini。

为什么这件事重要？

Minecraft 的魅力在于自由创造，但很多重复操作本质上只是准备工作或命令语法。

读取附近状态、拿物品、调时间、清雨、搭一个房间、替换一片墙、给房间补灯，这些事情都很常见。Blockwright 把这些操作变成一套受控的助手工作流，让玩家和运营把注意力放回正在创造的世界，同时保留明确、可追踪的执行过程。

一眼看懂

Blockwright 是一个能听懂自然语言、也能调用受控工具的 Minecraft AI 助手：你提出操作意图，它把重复、繁琐、命令化的工作变成游戏里的实际动作。

它面向 AI 操作 Minecraft 的工作流：读取上下文、执行支持动作、建造、改造、记录，并校验结果。
