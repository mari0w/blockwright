我给 5 岁女儿做了一个 Minecraft AI 模组：她不会认字，也能开口玩我的世界

我最近在做一个 Minecraft Java 版模组，名字叫 Blockwright。

它的起源很简单，不是什么宏大的 AI 改造游戏计划，而是我 5 岁的女儿。

她很喜欢玩《我的世界》，也知道很多东西应该怎么说。比如她知道自己想要火把、床、玻璃、门，也知道想让世界变成白天，或者想让旁边出现一个小房子。

但问题是，她识字不多。

Minecraft 里很多道具、菜单、命令和操作，对成年人来说只是“找一下”“搜一下”“输一下”，对一个 5 岁小朋友来说就不一样了。她可能知道“我要火把”，但不认识物品栏里的字；知道“我要盖房子”，但不知道从哪里开始；知道“我想把这里改一下”，但不知道应该怎么表达成游戏里的操作。

所以我当时想，既然她已经会说，能不能让 Minecraft 也听她说？

这就是 Blockwright 最早的想法：把自然语言和语音，变成 Minecraft 里的真实操作。

它不是一个“聊天机器人”

Blockwright 装好之后，玩家可以通过几种方式发需求：

  • 在 Minecraft 聊天栏输入 /bw ...
  • 打开本地或局域网 Web 页面打字
  • 在 Web 页面按住麦克风说话
  • 接入 Element/Matrix、钉钉或本地脚本等聊天入口

比如可以直接说：

给我一组火把
把时间调到白天
不要下雨了
帮我盖一个有窗户、床和灯的小木屋
把我面前这面墙换成石砖

对孩子来说，重点不是“AI 很厉害”，而是她终于不用先认识所有道具名字、菜单文字和命令格式，才能表达自己想做什么。

她可以先说“给我火把”，拿到火把之后再自己去插；也可以说“帮我盖一个小房子”，然后在 AI 搭出来的基础上继续改，拆一点、加一点、换材料、装饰一下。

这比直接生成一个完整作品更有意思。孩子不是被 AI 替代，而是多了一个能帮忙准备材料、搭基础、做重复操作的助手。

我最看重的几个场景

第一个是 找物品和给物品。

小朋友经常知道自己想要什么，但不会在创造模式物品栏里搜索。Blockwright 可以把“给我一把钻石剑”“给我一些玻璃”“给我一张床”变成实际的物品发放，并尽量让物品出现在玩家主手可见的位置。

第二个是 常见世界操作。

比如白天、晴天、夜视、创造模式、传送、效果、经验、召唤等，本质上都可以通过 Minecraft 命令完成，但普通玩家没必要记命令格式。用自然语言说出来会更顺手。

第三个是 建筑起步。

很多人玩 Minecraft 的时候，不是不想建，而是不知道第一步怎么下手。Blockwright 可以先搭一个能玩的结构：地板、墙、屋顶、入口、两格高室内空间、床、照明、窗户和可达路径。玩家再基于这个结构继续二次创作。

第四个是 继续修改。

比如先说：

帮我盖一个小木屋

再接着说：

把屋顶加高一点
多加几扇窗
里面放一张床和几盏灯
把这面墙换成石砖

这种交互更接近真实建造过程。先有一个雏形，然后边看边改。

为什么我把它做成局域网 Web 端

我自己的使用方式是本地 Minecraft 世界加局域网 Web 页面。

在游戏里输入：

/bw web

就可以打开 Blockwright 的 Web 地址。本机一般是：

http://127.0.0.1:8765/web

局域网里也可以让手机、平板或另一台电脑访问对应地址。这样小朋友可以不切出游戏，也可以用旁边的设备按住麦克风说话。

这个设计对家庭场景很实用：Minecraft 还在原来的存档里跑，不需要把地图搬到一台专门的 Paper 服务器，也不需要孩子理解什么是命令行、API key 或后端服务。

Blockwright 的主安装方式是 Fabric 模组，面向 Minecraft Java Edition 1.21.x、单人存档和局域网开放世界。Paper 插件也保留着，但主要给独立服务器场景使用。

它背后的工作流

Blockwright 大致分成两部分：

  • Minecraft 侧的 Fabric 模组负责读取玩家状态、扫描附近方块、发物品、放置方块、执行命令和回写结果。
  • 本地 controller 负责 Web 页面、模型配置、任务队列、蓝图、构建记录、MCP 工具和 AI 规划。

我没有把它做成一个“AI 随便执行命令”的黑盒。尤其是建筑任务，它会先保存蓝图和构建记录，再把同一份方块清单交给 Minecraft 执行端。执行端放置后还会逐块读取世界状态，生成校验报告。

简单说就是：自然语言进来，受控动作出去，结果还能检查。

这对我来说很重要。因为 Minecraft 不是一张图片，最终是否成功，要看世界里方块真的有没有被放对、物品有没有真的到玩家手上、命令有没有真的执行。

当前支持什么

目前 Blockwright 支持的入口包括：

  • Minecraft 内 /bw 命令
  • 本地 Web 文字聊天
  • Web 语音输入
  • Element/Matrix
  • 钉钉机器人
  • 本地命令或自定义脚本入口

支持的主要能力包括：

  • 读取玩家状态：主手、副手、快捷栏、背包
  • 扫描玩家附近世界方块
  • 发放物品，并尽量切到玩家主手可见位置
  • 调整时间、天气、效果、模式等命令类操作
  • 通过蓝图放置建筑
  • 基于已有构建记录继续改造
  • 保存蓝图和构建记录
  • 执行后回写校验报告

支持的模型后端包括：

  • Codex CLI
  • OpenAI
  • DeepSeek
  • 豆包 Doubao
  • Gemini

它现在还不是“万能 Minecraft AI”

我不想把它宣传成万能工具。

Blockwright 现在更适合这些场景：

  • 孩子或新手用语音、文字完成常见操作
  • 创造模式里快速拿材料、调环境、做建筑起步
  • 单人存档或局域网开放世界里的本地助手
  • 想把 Minecraft 接到 Web、语音或聊天工具里的玩家和服主
  • 需要可记录、可校验的 Minecraft AI 执行链路的开发者

它还在持续迭代。比如更复杂的建筑改造、撤销/回滚、图片到蓝图、更清晰的异常报告，这些都还可以继续做。

但对我女儿来说，最早那个问题已经被解决了一部分：她可以不用先学会所有字和所有命令，也能更自然地表达“我想要什么”。

这就是我觉得它有价值的地方。

项目地址

Blockwright 已经开源：

GitHub：https://github.com/mari0w/blockwright

如果你也在玩 Minecraft Java 版，或者你家里也有一个很会说、但还不太会认字的小玩家，可以试试看这个方向。

对我来说，Blockwright 不是为了让 AI 替孩子玩游戏，而是希望它能像一个懂 Minecraft 的小助手：帮她拿到道具、搭起第一面墙、点亮房间，然后把真正有创造力的部分留给她自己。
