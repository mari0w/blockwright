# Code of Conduct

Blockwright aims to be a practical, welcoming project for people building local-first Minecraft automation tools.

## Expected Behavior

- Be respectful and concrete when discussing issues, design tradeoffs, and implementation details.
- Assume contributors are acting in good faith, but keep technical feedback specific and evidence-based.
- Keep player safety, server safety, and credential safety visible when proposing features.
- Use inclusive language and avoid personal attacks, harassment, or repeated disruptive behavior.

## Unacceptable Behavior

- Harassment, threats, insults, or discriminatory language.
- Publishing private information, tokens, server addresses, or chat credentials without consent.
- Encouraging users to bypass authentication or expose unsafe controller endpoints.
- Repeatedly derailing technical discussions after maintainers ask for focus.

## Enforcement

Maintainers may edit or hide comments, close issues, restrict participation, or block users when needed to keep the project usable and safe.

Report conduct concerns through the maintainers' private contact channel or GitHub moderation tools. Security vulnerabilities should follow [SECURITY.md](SECURITY.md).
